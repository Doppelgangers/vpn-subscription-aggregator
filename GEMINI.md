# Правила работы с проектом

- Всегда использовать виртуальное окружение: `source env/bin/activate`.
- Использовать Conventional Commits (feat:, fix:, chore:, и т.д.).
- НИКОГДА не используем базовую админку Django для бизнес-логики. Все действия должны быть вынесены в самописные эндпоинты и кастомный UI.
- Использовать единый дизайн-код (описан в PLAN.md).
- Очереди задач: Celery + Redis.
- Весь новый код должен быть типизирован (type hinting).
- Код должен соответствовать PEP8.
