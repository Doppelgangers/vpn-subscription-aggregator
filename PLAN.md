# План и Техническое описание Хаба

## Суть проекта
Приложение работает как прокси-агрегатор подписок. Оно позволяет пользователю вставить ОДНУ ссылку в VPN-клиент, а на стороне сервера гибко управлять списком реальных серверов (нод).

## Модели данных
- `AggregateSubscription`:
    - `name`: Внутреннее имя (для админки).
    - `client_title`: Публичное имя (заголовок подписки в приложении).
    - `token`: Уникальный UUID или строка для URL.
    - `custom_base_url`: Домен для генерации красивых ссылок.
- `SourceLink`:
    - `url`: Прямая ссылка на подписку от 3x-ui.
    - `remark`: Краткое описание ноды.
    - `is_active`: Флаг включения в общую выдачу.
    - `config_count`: Кэшированное кол-во найденных ключей.

## Логика обработки запроса (aggregate_sub_view)
1. **Fetch:** Выполнение GET запросов ко всем URL из списка.
2. **Traffic Parse:** Чтение HTTP заголовка `Subscription-Userinfo` от нод. Суммирование байтов.
3. **Config Parse:** Декодирование Base64 тела ответа. Извлечение строк, начинающихся с `vless://`, `vmess://`, `trojan://`, `ss://`.
4. **Header Construction:**
    - `profile-title`: Установка имени в формате `base64:XXXXX` для поддержки кириллицы.
    - `Subscription-Userinfo`: Формирование сводной статистики трафика.
5. **Response:** Выдача объединенного списка в Base64.

## Дизайн и UI/UX
- **Схема:** Slate 800 (Dark Nav), Blue 600 (Primary buttons), Slate 50 (Background).
- **Компоненты:**
    - Toast-уведомления для подтверждения действий (копирование, удаление).
    - Кастомные модальные окна для подтверждения удаления.
    - Живая проверка статуса ссылок прямо в интерфейсе.
